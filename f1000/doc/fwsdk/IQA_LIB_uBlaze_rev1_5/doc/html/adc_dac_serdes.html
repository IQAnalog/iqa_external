ADC Capture to SerDes TX and SerDes RX to DAC Output

This section describes the procedures for the end-to-end data capture from ADC, through the DSP, and arriving at SerDes TX for trasmit. And for the opposite direction, end-to-end data capture from an external source (such as FPGA) arriving and captured by SerDes RX, through the DSP, and output from the DAC. Data can be then viewed using a spectrum analyzer.

To initialize the ADC and DAC in F1000, the following API can be called:

iqa_init_analog();

RISC-V (Embedded Controller, or ec) will then be initialized with the calibration firmware using the following API:

iqa_ec_load_firmware();

To initialize SerDes TX and RX, call the following API:

iqa_serdes_init();

Enabling/Disabling JESD TX lane can be achieved by calling API:

iqa_jesd_tx_lane_enable();

Calling to the following API will assist SerDes to achieve DLL lock:

iqa_help_dll_lock();

Next step is to setup the ADC DSP with the following API:

iqa_init_adc_dsp();

Corresponding API for the DAC DSP is the following API:

iqa_init_dac_dsp();

The user can specify desired DSP frequencies by invoking these APIs for ADC and DAC respectively:

iqa_set_adc_dsp_freq();

iqa_set_dac_dsp_freq();

Disable the SerDes RX link in preparation for further initialization by calling API:

iqa_jesd_rx_link_enable();

To prepare OOB processor, the following API can be called:

iqa_oob_sw_reset();

Merlin core should be programmed using the following API:

iqa_merlin_load_firmware();

Enabling the SerDes TX link can be achived by calling this API:

iqa_jesd_tx_link_enable();

Finally enabling the SerDes RX link by calling API:

iqa_jesd_rx_link_enable();

To select ADC DSP freq index, how many bands to user, as well as rate mode can be achieved by calling API:

iqa_adc_dsp_enable();

Corresponding API for DAC DSP is::

iqa_dac_dsp_enable();

In order to synchronize among ADCs in a F1000 as well as across multiple F1000s, SYSREF request for 3 pulses must be generated. These pulses indicate SOF of the multi-frame serial data for synchronization with the FPGA.

Method to generate the SYSREF request is implementation dependent. Refer to the example codes on how this is achieved with a F1000 EVM board.
