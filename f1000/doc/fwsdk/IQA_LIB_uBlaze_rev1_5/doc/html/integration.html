IQA F1000 SDK
Integration Guide

The linkable, pre-compiled library file(s) in the 'lib' directory have been compiled for the target processor identified by the release directory using the GNU toolchain (gcc) in Little Endian format. It is anticipated that the client application will also be developed using the gcc toolchain. Integrating the library into the target application requires the following steps:

  • Ensure the 'include' directory is accessible by the gcc compiler by adding a CFLAGS directive such as:

    CFLAGS += -I {path_to_release}/include

  • Ensure the 'lib' directory is accessible by the gcc linker by adding an LDFLAGS option sucs as:

    LDFLAGS += -L {path_to_release}/lib

  • Include the 'lib_iqa_api.a' file in the gcc link command:

    LIBS += lib_iqa_api.a

  • In any file which makes calls to the library, include the main header file. This file will include all other header files within the 'include' directory.

include "iqa_f1000_api.h"

  • Provide a hardware specific implementation of the Hardware Abstraction Layer (HAL). It is beyond the scope of this document to describe how the hardware specific HAL should provide physical access to the F1000 SPI port. Instead, the source code for a sample HAL has been provided, and IQA will work with the client's firmware engineers to assist with the development of a HAL.

  • Perform a simple test by calling the 'iqa_read_chip_id()' function and performing a compile and link. For F1000 REV A, the chip ID should read as 0x0F100001 whereas REV B silicon will return 0x0F100002.

NOTE: The library in this SDK does make a few limited calls to external standard library routines, such as snprintf, etc. It is anticipated that minimal stdlib routines are available.
