IQA F1000 SDK
Release History

This section tracks the details of the key changes, issues fixed and any known issues for all released versions of the IQA SDK.

Version 1.5

July 9, 2020

Key Changes and Issues Fixed

  1. Added 4 SerDes RX lanes support for F1000 revision B silicon for rate mode 2Gsps. Only EP-B DAC-1 is tested.
  2. Modified ADC scaler ratio to 1.4.
  3. Added new APIs iqa_init_dac_dsp(), iqa_set_dac_dsp_rate_mode(), iqa_set_dac_dsp_freq(), iqa_dac_dsp_enable() to configure DAC DSP.
  4. An additional parameter has been added to testapp sub-commands "nco_freq" and "nco_enable" for the configuration of the DAC parameters.
  5. Added testapp sub-command "adc_dac_serdes" for an end-to-end demo.
  6. Corrected ADC timing cal issue in RISC-V calibration firmware.
  7. Updated RISC-V v3.3.0 calibration firmware.
  8. Updated Merlin firmware to work with "adc_dac_serdes" demo.
  9. Updated F1000 rev B CSV tar file.
  10. Added sample code for ADC-DAC-SERDES application under /examples/xilinx/adc_dac_serdes directory.
  11. Updated client application source code at src/f1000_client directory

Note

  1. Only 4 SerDes RX lanes are enabled in this release.

Known Issues

  1. If both ADCs are brought up at the same time, only ADC-2 is functional. This applies to both Element Processors. ADC-1 for either Element Processor, however, is functional when initialized on its own.

Version 1.4d

June 19, 2020

Key Changes and Issues Fixed

  1. Added SerDes RX support for F1000 revision B silicon with rate mode 2Gsps. Only EP-B ADC-2 is supported currently.
  2. Added new API iqa_adc_dsp_enable() to configure DSP.
  3. Added testapp sub-command "nco_enable" to allow user to switch various parameters such as rate mode and frequency index.
  4. Added testapp sub-command "adc_dsp_serdes" for demo purposes.
  5. Updated Merlin firmware for 2Gsps rate mode.
  6. Added sample code for ADC-NCO-SERDES application with 2Gsps rate mode under /examples/xilinx/adc_nco_serdes_2gsps directory.

Note

  1. DVDD voltage should be set to 0.85V for F1000 rev B on rev B EVM for ADC and DAC operations.
  2. Higher DVDD voltage (0.94V) is required when SerDes RX lanes are enabled.
  3. Only 1 SerDes RX lane should be enabled in this release.

Known Issues

  1. If both ADCs are brought up at the same time, only ADC-2 is functional. This applies to both Element Processors. ADC-1 for either Element Processor, however, is functional when initialized on its own.

Version 1.4c

June 12, 2020

Key Changes and Issues Fixed

  1. Added SerDes RX support for F1000 revision B silicon. Only EP-B ADC-2 is supported currently.
  2. Improved F1000 revision B bring up initialization.
  3. Added iqa_get_cal_status() to retrieve ADC calibration completion status.
  4. Added iqa_help_dll_lock() API to assist DLL to lock for ADC, DAC and SerDes.
  5. An additional parameter has been added to iqa_set_adc_dsp_freq() API.
  6. Updated testapp application to download corresponding ADC calibration firmware for F1000 revision B.
  7. Added sub-command "adc_cal_status" to testapp application.
  8. Added sub-command "nco_freq" to testapp application which allows user to set desired NCO frequencies.
  9. Added Element Processor option for f1000_load_firmware command.
  10. Added mutex in HAL layer of the SPI transactions. (See src/hal/)
  11. Updated RISC-V firmware to v3.2.0 which has reduced the calibration completion time to 1 second.
  12. Updated Merlin firmware.
  13. Reduced MicroBlaze API library size.
  14. Updated "client" application with source code at "src/f1000_client" directory.
  15. Updated "binaries/fasm" tool with source code at "src/fasm" directory.
  16. Added sample code for ADC-NCO-SERDES application under /examples/xilinx/adc_nco_serdes directory.

Note

  1. This release has been tested with F1000 revision B silicon on IQ-Analog EVM only.
  2. DVDD voltage should be set to 0.85V for F1000 rev B on rev B EVM for ADC and DAC operations.
  3. Higher DVDD voltage (0.94V) is required when SerDes RX lanes are enabled.
  4. Only 1 SerDes RX lane should be enabled in this release.

Known Issues

  1. If both ADCs are brought up at the same time, only ADC-2 is functional. This applies to both Element Processors. ADC-1 for either Element Processor, however, is functional when initialized on its own.

Version 1.4b

April 27, 2020

Key Changes and Issues Fixed

  1. Added support for F1000 revision B silicon (ADC only). (This release has been tested with F1000 revision B silicon only.)
  2. Example program "adc_demo" has been modified for added support of revision B silicon.

Version 1.4a

April 15, 2020

Key Changes and Issues Fixed

  1. Added support for F1000 revision B silicon (DAC only). (This release has been tested with F1000 revision B silicon only.)
  2. Updated client program to support revision B silicon.
  3. Updated csv files for F1000 revision B.
  4. Example program "dac_demo" has been modified for added support of revision B silicon. The code has not been tested.

Version 1.4

March 06, 2020

Key Changes and Issues Fixed

  1. New API for Merlin Embedded Trace Buffer with example code.
  2. Added check for revision compatibility for the ALD file to be download into F1000 RISC-V.
  3. Updated elf2ald tool to add compatibility information for F1000 revision A and B.
  4. Added csv files for F1000 revision B.
  5. Added PHY I/O trace buffer support in Merlin debugger for F1000 revisoin B.

Known Issues

  1. For F1000 revision A, only DAC2 of Element Processor B can be initialized and brought up.

Version 1.3

January 10, 2020

Key Changes and Issues Fixed

  1. Added procedures for end-to-end ADC to SerDes TX with examples.
  2. New API to set RF synthesizer output power.
  3. API starts with iqa_ in function name.
  4. API returns negative value for error conditions.
  5. Updated Merlin "canned" firmware.
  6. Added Merlin "ramp" firmware.
  7. Resolved file format error in file tx_canned.ald when download into Merlin.
  8. Resolved iqa_ec_load_firmware() not returning error if file signature check fails.
  9. Added support for F1000 EVM rev B board.

Known Issues

  1. Only DAC2 of Element Processor B can be initialized and brought up.

Version 1.2

July 12, 2019

Key Changes and Issues Fixed

  1. Added revision history to documentation.
  2. Added examples code to SDK.
  3. Resolved iqa_ec_load_firmware() not working in Xilinx MicroBlaze environment.
  4. Added API to read chip revision.
  5. Allocate dynamic memory only when needed.
  6. Added Government Purpose Rights heading to API source header files.
  7. Updated RISC-V ADC calibration firmware.

Known Issues

  1. Not all 4 ADCs can be brought up at the same time. Only ADC2 and DAC2 of Element Processor B are being initialized.

  2. Added feature to DAC and ADC memory capture and playback.

  4. Fixed issue in Testapp while setting the Clock generator output amplitude.
  5. Addition of FFT plot and time domain plot in client program.
  6. Updated the iqa_set_clk_term() function.
  7. Resolved missing reserve_bytes() function causing unresolved reference during link.
  8. Updated documentation and corrected Doxygen formatting errors.

Known Issues

  1. Not all 4 ADCs can be brought up at the same time. Only ADC2 and DAC2 of Element Processor B are being initialized.
