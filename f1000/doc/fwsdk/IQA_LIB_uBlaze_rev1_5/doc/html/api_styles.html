IQA F1000 SDK
API styles

There are 2 types of API calls – register access APIs and mailbox APIs.

Register access APIs allow direct read/write access to the registers in F1000. When appropriate, direct memory read/write APIs are provided and recommended
to be used for efficiency reason.

The second method, through mailbox, will support saving data to the mailbox’s incoming area. Embedded controller is being notified that there are data to be
processed. When the embedded controller completes processing, it will generate an interrupt as notification that the processing is finished and the result
and status are available to be read by the caller.

The APIs are categorized to be blocking and non-blocking calls. The non-blocking APIs may be useful in an RTOS environment where OS can continue process
its tasks without the need to wait for the API to finish. A notification will be generated when the API completes its execution.
