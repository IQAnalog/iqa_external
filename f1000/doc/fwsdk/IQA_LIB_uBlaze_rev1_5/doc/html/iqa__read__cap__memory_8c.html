Reads data from the capture memory of F1000.

Data read from the ADC/DAC capture memory will be 16-bit packed in Little Endian format. Samples must be read in multiples of 64 sample "chunks". A chunk can be the entire 32K sample memory. This routine currently calls "malloc" to allocate a working buffer twice the size of the samples for re-ordering the data read from the SRAM.

Parameters
f1000_id: An Integer that specifies the F1000 ID - Default 1
ep_id: Value that represents the Element Processor ID.
Element Processor Value
A 0x1
B 0x2
mem_id: Value that specifies the memory type ID. Following are the values for various Memory types
Memory Type Value
ADC/DAC 1 memory in 10-bit mode 0
ADC/DAC 2 memory in 10-bit mode 2
Test 1 memory 4
Test 2 memory 5
offset: An integer that specifies the Offset of memory where the data needs to be read. This should start off at zero and increment on each successive call by the total nuumber of samples previously read.
ordered: integer specifying if the data is ordered. Should be set to 1 unless you really understand the format of un-ordered data.
buf: Pointer to the data buffer, which contains the the data that is read from F1000 capture memory.
samples: An integer specifying how many samples to be read. Each sample size is 16-bits.
Returns
: OK on Success else ERROR
