SPI Interfaces

F1000 supports both master and slave SPI interfaces. All communication with the F1000 from the client HOST processor is done using the slave SPI interface.

SPI is a serial interface which uses either 4 wire (Standard) or 6 wires (Quad SPI). The Standard Wires are : SPI_CLK, SPI_CS_N, MOSI (Master Out Slave In), MISO (Master In Slave Out. The Quad SPI has 2 extra wires called Data3, Data2.

The SPI Interface works in terms of  byte transfers. The Write operation requires 9 bytes and the read operation requires 14 bytes.  Typically you have
SPI Write and Read byte buffers. Data sent to the slave is put in Write Buffers and Data Slave returned is read back from the Read Buffers. The SPI
slave follows the following Protocol :  One Byte Command (Read/Write) followed by Address (4 Bytes) and Data (4 bytes). For Read’s the Data is normally
sent as 0 and the Read takes extra cycles for the chip to do a fetch and return the results. Actual operation is typically some “SPI_CMD (WriteBuffer, ReadBuffer, cnt_of_bytes_to_be_sent)” .

Multiple Data can be transferred after the single transfers defined below by keeping the CS_N low. The address supplied below initially will automatically
increment by 4 after each Data transferred. This can be used for code download as well as communication to the modules on the interface.

SPI Write Register/Memory operation

WriteBuffer[0] = 0x02; // Write command
WriteBuffer[1] = addr [31:24];
WriteBuffer[2] = addr [23:16];
WriteBuffer[3] = addr [15:8];
WriteBuffer[4] = addr [7:0];
WriteBuffer[5] = data [31:24];
WriteBuffer[6] = data [23:16];
WriteBuffer[7] = data [15:8] ;
WriteBuffer[8] = data [7:0];

SPI_CMD (WriteBuffer, ReadBuffer, 9);

SPI Read Register/Memory operation

WriteBuffer[0] =   0x0B; // Read command
WriteBuffer[1] =   addr [31:24];
WriteBuffer[2] =   addr [23:16];
WriteBuffer[3] =   addr [15:8];
WriteBuffer[4] =   addr [7:0];
WriteBuffer[5] =   0;
WriteBuffer[6] =   0;
WriteBuffer[7] =   0;
WriteBuffer[8] =   0;
WriteBuffer[9] =   0;
WriteBuffer[10] =  0;
WriteBuffer[11] =  0;
WriteBuffer[12] =  0;

SPI_CMD (WriteBuffer, ReadBuffer, 13);

After the above command, you will receive The Read data 32 bits in ReadBuffer[9] to ReadBuffer[12].

Quad SPI

F1000 also supports Quad SPI for both Master and Slave. The chip starts in normal SPI mode, but can be switched over to Quad SPI, after sending a
command initially over normal SPI. In Quad SPI mode, the data transfer is bi-directional using all the available 4 lines - D3-MSB, D2, MISO (D1)
and MOSI (D0)-LSB. So it is 4 times faster than normal SPI.  The switch to Quad SPI mode is as follows:

WriteBuffer[0] =   0x01; // Command to switch to Quad mode
WriteBuffer[1] =   0x01;
SPI_CMD (WriteBuffer, ReadBuffer, 2);
