IQA F1000 SDK
Examples

The purpose of these examples is to demonstrate which APIs and in what sequence to call in the SDK library.

Note
Example code in these directories should be compiled with the files in the "common" directory and linked with the IQA SDK library. The output file can be downloaded into the target platform for execution.

Hardware Platform

The following boards are configured to execute the examples:

  1. Xilinx Virtex UltraScale+ FPGA VCU118 Evaluation Kit
  2. IQ-Analog F1000 EVM

Connections between Xilinx FPGA and F1000 EVM are wires for the SPI interface with Xilinx FPGA as SPI master and F1000 as slave.

Programming Environment

Xilinx Vivado SDK version 2018.2 (64-bit). Xilinx VCU118 is configured to run MicroBlaze processor in Little Endian mode.

Operation

Power rails to the F1000 should be turned on before any communications to take place.

Example List

adc_demo

This example demonstrates the procedures and the API calling sequence to initialize the ADC block, and display the captured data in the ADC memory.

adc_dac_serdes

This example demonstrates the procedures and the API calling sequence to initialize the ADC block including the DSP, DAC block including the DSP, and both SerDes TX and RX. It includes supporting commands to change the NCO frequency (ADC and DAC) from an external controller such as FPGA. Data from the ADC path can be captured at FPGA for processing, such as plotting. Output from DAC can be observed with a spectrum analyzer.

adc_nco_serdes

This example demonstrates the procedures and the API calling sequence to initialize the ADC block, including the DSP, capture the data and send to SerDes TX. It also supports commands to change the NCO frequency from an external controller such as FPGA. Data can be captured at FPGA for processing, such as plotting.

adc_nco_serdes_2gsps

This example is an extension to adc_nco_serdes example that it demonstrates the procedures for the operation in 2Gsps rate mode.

adc_serdes_tx

This example demonstrates the procedures and the API calling sequence to initialize the ADC block, capture the data and send to SerDes TX. Data can be captured at FPGA for processing, such as plotting.

chipid

This example demonstrates how to call the API to read chip ID and revision from F1000.

common

These are core files to be compiled and linked with the example source files to generate the executable.

common/hal_spi.c : Sample HAL implementation of the SPI functions for the MicroBlaze environment.
common/spi_header.h : Header file for SPI interface from Xilinx.
common/xspi.c : SPI implementation for Xilinx FPGA.

dac_demo

This example generates a sine wave and stores the data in the capture memory, to be playback by the DAC. Steps on initializing the chip and related APIs are shown.

merlin_fw

This example shows the API used to download firmware into the Merlin processor.

riscv_fw

This example shows the API used to download firmware into the RISC-V processor.

sdkrev

This example demonstrates the API call to read the SDK revision.

serdes_tx

This example shows how to load data pattern into memory and send the memory data to SerDes TX. FPGA can capture the SerDes TX data for further processing.
